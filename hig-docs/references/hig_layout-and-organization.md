# Layout and organization

- [Boxes](/design/human-interface-guidelines/boxes) - A box creates a visually distinct group of logically related information and components.
- [Collections](/design/human-interface-guidelines/collections) - A collection manages an ordered set of content and presents it in a customizable and highly visual layout.
- [Column views](/design/human-interface-guidelines/column-views) - A column view — also called a  — lets people view and navigate a data hierarchy using a series of vertical columns.
- [Disclosure controls](/design/human-interface-guidelines/disclosure-controls) - Disclosure controls reveal and hide information and functionality related to specific controls or views.
- [Labels](/design/human-interface-guidelines/labels) - A label is a static piece of text that people can read and often copy, but not edit.
- [Lists and tables](/design/human-interface-guidelines/lists-and-tables) - Lists and tables present data in one or more columns of rows.
- [Lockups](/design/human-interface-guidelines/lockups) - Lockups combine multiple separate views into a single, interactive unit.
- [Outline views](/design/human-interface-guidelines/outline-views) - An outline view presents hierarchical data in a scrolling list of cells that are organized into columns and rows.
- [Split views](/design/human-interface-guidelines/split-views) - A split view manages the presentation of multiple adjacent panes of content, each of which can contain a variety of components, including tables, collections, images, and custom views.
- [Tab views](/design/human-interface-guidelines/tab-views) - A tab view presents multiple mutually exclusive panes of content in the same area, which people can switch between using a tabbed control.
